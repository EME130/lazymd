Installation

Prerequisites

lazy-md requires Zig version 0.15.1 or later.

Build from source

git clone https://github.com/user/lazy-md.git cd lazy-md zig build

The compiled binary is at zig-out/bin/lazy-md. Move it to your PATH:

cp zig-out/bin/lazy-md /usr/local/bin/

Pre-built binaries

Download pre-built binaries from GitHub Releases. Available for:

  • Linux x86_64
  • macOS x86_64
  • macOS ARM64 (Apple Silicon)

Quick Start

# Open a markdown file lazy-md README.md # Open in current directory (shows file tree) lazy-md # Create a new file lazy-md notes.md

Once inside the editor:

  1. Press i to enter Insert mode
  2. Type your markdown content
  3. Press Escape to return to Normal mode
  4. Type :w and Enter to save
  5. Type :q and Enter to quit

Your First File

lazy-md supports standard Markdown (.md) and the custom .rndm format (which is 100% compatible with Markdown).

  • Left panel â€” File tree showing the current directory

Editing Modes

lazy-md uses vim-style modal editing with three modes:

Normal Mode

The default mode. Used for navigation, commands, and switching to other modes. The status bar shows NORMAL.

Insert Mode

For typing text. Enter with i, a, o, or O. Exit with Escape. Status bar shows INSERT.

Command Mode

For executing commands. Enter with :. Status bar shows COMMAND.

Mouse Support

lazy-md supports mouse input in terminal emulators that support SGR extended mouse mode (most modern terminals):

  • Left click â€” Position cursor in the editor, or switch focus to clicked panel
  • Scroll wheel â€” Scroll the editor content up/down (3 lines per scroll)

Mouse support is automatically enabled when the editor starts and disabled when it exits.

File Formats

Plugin System Overview

lazy-md has an extensible plugin architecture that lets you:

  • Register custom commands
  • Hook into editor events (file open/save, buffer changes, mode changes)
  • Add custom UI panels
  • Extend the status bar

Plugins are Zig modules that implement the Plugin interface using a vtable pattern.

Developing Plugins

See the full Plugin Development Guide for detailed instructions.

Quick overview

  1. Create a Zig file in src/plugins/
  2. Implement the plugin interface methods: pluginInfo(), pluginInit(), pluginDeinit(), onEvent(), getCommands()
  3. Register your plugin in main.zig with plugin.makePlugin()
  4. Build and test

Example: Hello World Plugin

const plugin = @import("../plugin.zig"); const Self = @This(); pub fn pluginInfo(self: *Self) plugin.PluginInfo { _ = self; return .{ .name = "hello", .version = "0.1.0", .author = "You", .description = "Says hello", }; } pub fn pluginInit(self: *Self, editor: *Editor) void { _ = self; editor.status.set("Hello plugin loaded!", false); } pub fn pluginDeinit(self: *Self) void { _ = self; } pub fn onEvent(self: *Self, event: *plugin.PluginEvent) void { _ = self; _ = event; } pub fn getCommands(self: *Self) []const plugin.CommandDef { _ = self; return &.{}; }

System Design

lazy-md is built with clean module separation:

main.zig (Event Loop) â”œâ”€ Terminal.zig Raw mode, ANSI escape codes, colors â”œâ”€ Input.zig Key + mouse event parsing â”œâ”€ Editor.zig Vim modes, cursor, scroll â”‚ â”œâ”€ Buffer.zig Gap buffer, undo/redo, file I/O â”‚ â””â”€ markdown/syntax.zig Tokenizer + theme â”œâ”€ ui/Layout.zig 3-panel geometry â”œâ”€ ui/Preview.zig Rendered markdown preview â”œâ”€ Renderer.zig Double-buffered cell grid â””â”€ plugin.zig Plugin system

Key design decisions

  • Gap buffer â€” Efficient for insert/delete at cursor position
  • Double buffering â€” Diff-based rendering prevents flicker
  • Zero dependencies â€” Uses only Zig stdlib + POSIX
  • Vtable plugins â€” Type-erased interface for extensibility

How to Contribute

  1. Fork the repository
  2. Create a feature branch
  3. Make your changes
  4. Run zig build test to ensure all tests pass
  5. Run zig fmt src/ to format your code
  6. Submit a pull request

# Debug build zig build # Release build zig build -Doptimize=ReleaseSafe # Run directly zig build run -- myfile.md
